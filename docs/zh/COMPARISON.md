# 与 mpdf、tcpdf、dompdf、FPDF 的对比

将 `dskripchenko/php-pdf` 与 Packagist 上使用最广泛的四个 PHP PDF
库进行逐项特性对比。性能数据请见 [BENCHMARKS.md](BENCHMARKS.md)。

## 许可证

| 库                       | 许可证          | OEM／专有软件捆绑 |
|--------------------------|----------------|--------------------------|
| **dskripchenko/php-pdf** | **MIT**        | ✅ 无任何摩擦 |
| mpdf/mpdf                | GPL-2.0-only   | ❌ 需要随附 GPL 或购买商业许可 |
| tecnickcom/tcpdf         | LGPL-2.1+      | ⚠️ 静态链接存在一些细节问题 |
| dompdf/dompdf            | LGPL-2.1       | ⚠️ 同 tcpdf |
| setasign/fpdf            | 可重新授权       | ✅ 但 FPDI 与其他扩展是专有的 |

MIT 是 PHP 生态中最宽松的许可证。您可以在任何场景下使用该库，
包括闭源产品，且无需修改其源码或公开自己的代码。

| 特性                                    | php-pdf | mpdf | tcpdf | dompdf | FPDF |

| 底层命令式 API                          | ✅      | ⚠️    | ✅    | ⚠️      | ✅   |
| 流式构建器（Document → Section）        | ✅      | ❌   | ❌    | ❌     | ❌   |

对于复杂的 HTML 或 CSS（Flexbox、多栏、`@media`、浮动），
`dompdf` 和 `mpdf` 走得更远。对于商业文档（段落、表格、列表、
行内样式、标题），`php-pdf` 与之持平，且具有更好的性能。

## 何时选择哪一个

### 选择 **dskripchenko/php-pdf** 当
- 您要交付闭源 / OEM 产品并需要 MIT 许可。
- 您要生成商业文档（发票、报表、合同、表单、证书），且
  在意排版、条形码、AcroForm、签名或 PDF/A。
- 您需要纯 PHP 中覆盖最广的条形码支持。
- 您面向 PHP 8.2+，希望使用现代化、类型安全的代码。
- 您需要 PDF 2.0 特性（AES-256 R6）。
- 您在意输出体积与单次渲染延迟。

### 选择 **mpdf/mpdf** 当
- 它已经在您的技术栈中，迁移成本不划算。
- 您的项目与 GPL 兼容。
- 您需要更丰富的 CSS 覆盖（Flexbox、`@media`、复杂选择器）。
- 您偏好 mpdf 的文档与 Stack Overflow 资料生态。

### 选择 **tecnickcom/tcpdf** 当
- 您在维护一个已经使用它的遗留企业 PHP 应用。
- 您明确需要公钥加密（`/Filter /PubSec`）。
- 您能接受单个 3 万行文件以及 PHP 7.1+ 风格的代码。

### 选择 **dompdf/dompdf** 当
- 您需要渲染任意带浮动布局的 HTML/CSS —— 面向浏览器的内容，
  而非商业文档。
- 性能和内存不是关键（它会在排版前在内存里构建完整 DOM）。

### 选择 **setasign/fpdf** 当
- 您生成的是简单的发票或收据，且使用手动定位。
- 您希望依赖体积尽可能小。
- 您不需要 UTF-8、HTML、表格、图表、加密或签名。
