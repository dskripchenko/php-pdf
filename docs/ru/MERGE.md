# Чтение и объединение существующих PDF

Помимо генерации PDF с нуля, `dskripchenko/php-pdf` умеет **читать готовые
PDF-файлы** и **комбинировать** их: присоединять один документ к другому,
выбирать отдельные страницы и накладывать страницу одного документа на страницы
другого.

Всё — на чистом PHP под лицензией MIT: без FPDI и без внешних бинарников.

## Чтение PDF

```php
use Dskripchenko\PhpPdf\Pdf\Reader\ReaderDocument;

$doc = ReaderDocument::fromBytes(file_get_contents('report.pdf'));

echo $doc->pageCount();               // число страниц
foreach ($doc->pages() as $page) {
    printf("%.0f × %.0f pt\n", $page->width(), $page->height());
}
```

`ReaderDocument` резолвит объекты лениво, проходит цепочку перекрёстных ссылок
(классические таблицы, XRef-потоки и object-потоки) и переходит к полному скану
файла, если xref повреждён.

## Присоединение документов

`PdfMerger` склеивает страницы из одного или нескольких источников в новый
документ (в стиле pdftk):

```php
use Dskripchenko\PhpPdf\Pdf\Merge\{PdfMerger, PdfSource};

$bytes = PdfMerger::create()
    ->append(PdfSource::fromFile('cover.pdf'))
    ->append(PdfSource::fromFile('body.pdf'))
    ->append(PdfSource::fromFile('appendix.pdf'))
    ->toBytes();

file_put_contents('combined.pdf', $bytes);
// или: PdfMerger::create()->append(...)->toFile('combined.pdf');
```

`PdfSource` принимает путь к файлу или сырые байты:

```php
PdfSource::fromFile('/path/to.pdf');
PdfSource::fromBytes($binaryString);
```

## Выбор и переупорядочивание страниц

Передайте номера страниц (с 1) в нужном для вывода порядке:

```php
$bytes = PdfMerger::create()
    ->append(PdfSource::fromFile('a.pdf'), pages: [1, 3, 5])   // подмножество
    ->append(PdfSource::fromFile('b.pdf'), pages: [2, 1])      // переупорядочено
    ->toBytes();
```

Опустите `pages`, чтобы взять все страницы в порядке чтения. Каждая выходная
страница сохраняет геометрию источника (MediaBox, CropBox, поворот).

## Встраивание (штамп) страницы

`stamp()` рисует страницу одного документа поверх уже присоединённых — страница
источника становится переиспользуемым Form XObject. Подходит для водяных знаков,
бланков, подложек или размещения страницы как иллюстрации.

```php
$bytes = PdfMerger::create()
    ->append(PdfSource::fromFile('invoices.pdf'))          // базовые страницы
    ->stamp(
        PdfSource::fromFile('watermark.pdf'),
        page: 1,                                            // какая страница источника
        onPages: null,                                      // null = каждая выходная страница
        placement: Placement::fit(),
    )
    ->toBytes();
```

Целевые страницы задаются через `onPages` (с 1):

```php
->stamp(PdfSource::fromFile('logo.pdf'), page: 1, onPages: [1], placement: Placement::at(40, 40, 0.5))
```

Собственный контент базовой страницы сохраняется и рисуется первым; наложение
идёт поверх из чистого графического состояния.

## Аннотации и закладки

Аннотации страниц и оглавление документа (закладки) переносятся в результат
**по умолчанию**. Внутренние ссылки и назначения закладок — включая именованные
назначения — ремапятся на новые страницы. Ссылка/закладка, чья целевая страница
не попала в вывод, отбрасывается; внешние `URI`-ссылки сохраняются всегда.

```php
PdfMerger::create()
    ->append(PdfSource::fromFile('a.pdf'))
    ->withoutAnnotations()   // отключить перенос аннотаций
    ->withoutOutlines()      // отключить перенос закладок
    ->toBytes();
```

Виджеты полей форм (AcroForm) и popup-аннотации не переносятся.

## Позиционирование

`Placement` управляет тем, как встраиваемая страница (в пунктах) ложится на
целевую:

| Фабрика | Поведение |
|---|---|
| `Placement::fit()` | Масштаб по размеру с сохранением пропорций, по центру. |
| `Placement::stretch()` | Заполнить целевую страницу целиком (пропорции игнорируются). |
| `Placement::at($x, $y, $scale = 1.0)` | Левый-нижний угол в `($x, $y)`, масштаб `$scale`. |

Повёрнутые страницы источника (`/Rotate` 90/180/270) выпрямляются через
`/Matrix` формы, поэтому позиционирование работает в понятных прямых координатах.

## Зашифрованный вход

Зашифрованные источники прозрачно расшифровываются при чтении и переэмитятся в
результат **без шифрования**:

```php
PdfSource::fromFile('protected.pdf', password: 'secret');
```

Поддержка: RC4 (40/128 бит), AES-128 (AESV2) и AES-256 (V5 R5/R6). Пробуются и
пользовательский, и владельческий пароль. Пустой пароль (частый случай «только
ограничения владельца») работает без аргумента.

## Что понимает reader

- Классические таблицы `xref`, XRef-потоки (PDF 1.5+), гибридный `/XRefStm` и
  инкрементальные обновления (`/Prev`).
- Object-потоки (сжатые объекты).
- Фильтры: Flate (с PNG/TIFF-предикторами), LZW, ASCII85, ASCIIHex,
  RunLength. Фильтры изображений (DCT/JPX/CCITT/JBIG2) копируются как есть.
- Восстановление при повреждённом/отсутствующем xref сканированием заголовков.
- Разворачивание дерева страниц с наследованием MediaBox/CropBox/Rotate/Resources.

## Ограничения (v1)

Объединение пересобирает каждую страницу из её контента и ресурсов. Пока **не**
переносятся в результат:

- Интерактивные поля форм (AcroForm) и widget/popup-аннотации — отбрасываются.
- Структурные теги (Tagged PDF / PDF-A-соответствие результата).
- Потоки изображений и шрифтов копируются как есть, без перекодирования.

Аннотации и закладки с внутренними/именованными назначениями **переносятся** и
ремапятся — см. [Аннотации и закладки](#аннотации-и-закладки).

Это задокументировано, чтобы «поддерживает merge» не читалось как «сохраняет
всё». Полный объём и планы — в `docs/design/pdf-merge.md`.
